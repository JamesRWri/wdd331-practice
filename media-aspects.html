Aspect Ratio and Object Fit in Layout

By James Wright · June 2026

Images and video have a fundamental tension with grid and flex layouts. A track can be any size the browser computes, but a photograph has a fixed ratio of width to height. If the image stretches to fill its track, it distorts. If it shrinks to preserve its ratio, it leaves gaps. CSS provides two complementary tools to resolve this: aspect-ratio enforces a ratio on any element, and object-fit controls how replaced content fills its container without distortion. Together they make media behave predictably inside any layout context.

The Problem Without These Properties

When you place an image inside a grid cell, the browser stretches it to fill the cell by default because grid items stretch along the block axis. The image maintains its intrinsic width but its height gets forced to match the cell, distorting the aspect ratio. Setting width to 100% fixes the width, but then the height is determined by the image's intrinsic ratio, which means cells in the same row will have different heights if their images have different proportions.

Core Takeaways

The modern pattern that handles nearly every card thumbnail layout situation combines these values cleanly. By declaring full width, setting a fixed aspect ratio, and locking down cover style positioning, you prevent layout shifts entirely while letting the browser crop your assets beautifully without distortion.
